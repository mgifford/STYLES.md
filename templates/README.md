# Templates

Blank scaffolds for starting your own `STYLES.md`.

Copy the template that best fits your context. Replace `[PLACEHOLDER]` values
with your project's real information. Remove sections that do not apply.

## Available templates

| Template | Best for |
| :--- | :--- |
| [STYLES-template.md](../STYLES-template.md) | General purpose starting point |
| [formal-gov.md](./formal-gov.md) | Public-sector, compliance-heavy, high-trust environments |
| [tech-minimal.md](./tech-minimal.md) | Developer tools, internal docs, speed-focused teams |
| [low-bandwidth.md](./low-bandwidth.md) | Sustainability-first, low-resource, offline-capable projects |

## After copying a template

1. Move the file to your project root and rename it `STYLES.md`.
2. Replace every `[PLACEHOLDER]` with your project's real values.
3. Remove any section that does not apply to your project.
4. Reference the file from your `AGENTS.md` and LLM prompts.
5. Add [ACCESSIBILITY.md](https://github.com/mgifford/ACCESSIBILITY.md) and
   [SUSTAINABILITY.md](https://github.com/mgifford/sustainability.md) as companion standards.
